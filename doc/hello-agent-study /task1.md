## 1. 笔记
- 关于智能体到底是什么？ 智能体和大模型有什么区别？
  - 我的理解： 
    - 智能体是一个AI产品或者系统，它和大模型是两个物种。智能体可以使用大模型，它利用了大模型的推理能力，结合一些tools，它也有记忆能力和规划能力，更像是复仇者联盟中 钢铁侠的虚拟管家贾维斯的形象。
    - 而大模型只是能思考，能推理问题，但无法做出比较真实的行动。大模型更加的工具化一些。
  
  - 举例：和LLM对话：我想喝咖啡，给我买个咖啡吧。  LLM会回答，如果你想喝到咖啡，你要怎么做12345等等。但绝不可能给你去买咖啡；  如果和一个智能体对话：给我买个咖啡吧。   智能体会判断：主人要买咖啡，需要线上购买，需要我能为他下单，需要我下的咖啡单最好是他喜欢喝常喝的品类；我去工具库里去找能做这个事的工具，比如我可以调美团外卖的API，我就能调用它，去完成线上下单甚至付款的事情。然后主人真的收到了咖啡外卖。
  - **总之**：智能体 = 大模型（思考） + 工具（行动） + 记忆（上下文） + 规划（策略）
  - 我希望一个agent可以自主地帮我解决一些问题


- 关于agent的运行机制
  - 循环、循环 还是循环
    - agent真正执行一次工作，在代码里都是一个for循环。即代码都在这个for循环里，次数可能是若干次或者判定提前结束。
    - 为什么是循环呢？————因为在跟环境不断地进行交互。
  - 如何交互？
    - 通过感知——思考——行动——观察——感知.....的多次循环逐步地推进任务
    - 简单的理解就有个类比：比如一个嵌入式恒温系统，这不涉及大模型的推理，要解决的任务也很单一，但逻辑是一样的————该系统的温度传感器一直监控环境温度，外部环境变化的时候，传感器传到芯片中的温度变化了，芯片里的判定系统说环境低了（比如），它发送升温指令给它外部的温度系统（比如燃气），多烧一些，让该恒温系统给环境提供更多热能，维持环境还处在恒温的状态下。 同时监控环境温度有没有恢复如初，还是反而变高了。再根据这些进行调节。
  - LLM agent的运行机制，名词概念：
    - 感知 (Perception)	接收环境输入（用户指令/工具返回结果）	
    - 思考 (Thought)	LLM内部推理，包含规划和工具选择	
    - 行动 (Action)	执行具体操作（调用工具）	
    - 观察 (Observation)
